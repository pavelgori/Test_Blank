Этот пример познакомит вас с некоторыми важными особенностями построения шаблонов сообщений.

HTML редактор

Используйте встроенный HTML редактор для создания шаблона веб-версии сообщения - вручную или с помощью панели управления, которая находится над окном редактора. Она содержит несколько инструментов, способных улучшить процесс работы с шаблоном:

  • жирный шрифт, курсив и зачёркнутый текст
  • нумерованные и маркированные списки
  • таблицы, изображения и гиперссылки
  • контент из внешних источников
  • utm метки и пиксели отслеживания
  • переменные для динамического содержимого

контрольные группы

Вы можете использовать в сообщении контент, динамически загружаемый с вашего сайта или из базы данных. Есть несколько способов это реализовать:

{json.one}{json.two}{json.object.three}{json.four} контент

{json.text}

Таблицы данных

Контент можно подставлять из таблиц данных, просто настройте одну.

Интеграция с API

Вы также можете использовать динамический контент из свойства "content" API запросов, связанных с кампаниями.

Логические выражения

Вы можете использовать логические выражения, {if "j" like json.one} такие, как это, {else} или не можете, {end} {for array[" и" "можете использовать" "циклы!"]} {.} {else} и можете использовать циклы, даже с пустыми массивами! {end}

Это просто, как {for json.array} {.} {end}добавлять циклы и динамически обновляемый контент!

Циклы с динамическим контентом помогут вам отправлять подписчикам персонализированную информацию.

{for $index $item = json.more_useful_example}

{$item.left_tag} {$item.text} {$item.right_tag}

{else}

Если такие данные существуют.

{end}
